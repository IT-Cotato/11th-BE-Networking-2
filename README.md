## 📌 Scenario & Requirement

### 시나리오

코테이토 운영진 김감자는 매년 증가하는 동아리 지원자들의 서류를 관리하는 데 어려움을 겪고 있습니다. 지금까지는 지원서를 구글 폼으로 받아 엑셀 파일에 정리하고 있었지만, 지원자가 늘어남에 따라 관리가 복잡해졌습니다. 김감자는 지원자 정보를 체계적으로 관리하고, 다른 운영진들과 실시간으로 평가 결과를 공유할 수 있는 시스템이 필요하다고 느꼈습니다. 이에 김감자는 지원자 정보를 저장하고 관리할 수 있는 REST API를 개발하여 지원자 관리 프로세스를 개선하기로 했습니다.

### 사용 기술

- SpringBoot
- Spring Data JPA
- MySQL

## 📌 요구사항

코테이토 동아리 지원자 관리 시스템을 구현합니다.

### 지원 서류 정보

지원 서류는 다음과 같은 정보를 가지고 있습니다:

- 이름: 한글 2글자 이상, 10글자 이하여야 합니다.
- 지원 기수: 해당 지원자가 지원한 기수를 나타내는 1 이상의 정수값입니다.
- 나이: 22살 이상 30살 이하여야 합니다.
- 파트: 기획, 디자이너, 프론트엔드, 백엔드가 있습니다.
- 참여 적극성: 해당 지원자의 참여 적극성을 나타내는 0 이상 10 이하의 정수값입니다.
- 성장 의지: 해당 지원자의 성장 의지를 나타내는 0 이상 10 이하의 정수값입니다.
- 휴대폰 번호: 010으로 시작하는 11자리 문자열입니다(예시: 01012345678).
- 서류 제출 시간: yyyy-MM-dd hh:mm 형태로 표현합니다(예시: 2025-02-28 23:30).

### 1. 지원자 관리 시스템 설계

- Spring Data JPA를 활용하여 필요한 엔티티와 관련 클래스들을 설계하세요.
- 지원자 정보를 저장할 데이터베이스 스키마를 설계하세요.
- 필요한 Repository, Service, Controller를 구현하세요.

### 2. 지원자 서류 등록 기능

- 지원자의 서류를 등록하는 API를 구현하세요.
- 요청 본문(Request Body)은 다음 정보를 포함해야 합니다
    - 이름
    - 지원 기수
    - 나이
    - 파트
    - 참여 적극성
    - 성장 의지
    - 휴대폰 번호

### 3. 지원자 서류 좋아요 기능

- 지원자 서류에 좋아요를 추가하는 API를 구현하세요.
- 좋아요 처리 시 권한 처리는 구현의 편의를 위해 생략합니다.

### 4. 지원자 서류 목록 조회 기능 (페이징, 필터링)

- 지원자 서류 목록을 페이징, 필터링하여 조회하는 API를 구현하세요.
- 기본 필터링 기준은 오래된 지원 시간 순입니다.
- 필터링 기준 옵션:
    - 지원 시간 오래된 순
    - 지원 시간 최신 순
    - 좋아요 높은 순 (좋아요 수가 같을 경우 지원시간 오래된 순)
    - 참여 적극성 높은 순(같은 경우 지원시간 오래된 순)
    - 성장 의지 높은 순(같은 경우 지원시간 오래된 순)
- 목록 조회 시 각 지원자 항목에는 다음 정보가 포함되어야 합니다:
    - 지원자의 식별번호
    - 지원 기수
    - 파트
    - 참여 적극성
    - 성장 의지
    - 받은 좋아요 수
    - 서류 제출 시간
- 페이지네이션 정보(현재 페이지 번호, 페이지 크기, 정렬 기준, 전체 요소 수, 전체 페이지 수)도 함께 반환해야 합니다.

### 5. 지원자 서류 세부 조회 기능

- ID를 통해 단일 지원자 서류를 조회하는 API를 구현하세요.
- 응답에 반드시 포함되어야 하는 필드는 다음과 같습니다.
    - 지원자의 식별번호
    - 지원 기수
    - 파트
    - 참여 적극성
    - 성장 의지
    - 받은 좋아요 수
    - 휴대폰 번호
    - 서류 제출 시간

### 6. 예외 처리

- 모든 API는 적절한 HTTP 상태 코드와 함께 응답해야 합니다.
- 유효하지 않은 요청에 대해 적절한 에러 메시지를 반환해야 합니다.

### Q & A

- 기존 코드를 수정해도 되나요? → 디렉토리 구조, 코드 등 모두 수정가능합니다.
- OM 분들은 자신의 스타일로 바꿔서 구현해주세요. 신입 부원분들이 최대한 다양한 코드를 보실 수 있는 기회라고 생각합니다.
- 요구사항에 문제가 있는 것 같아요 -> 11기 백엔드 카톡방에 올려주세요. 요구사항에 대한 역질문을 잘 하는 개발자가 되셨으면 좋겠습니다.
    - 예시: 좋아요는 취소가 가능한가요?
